# authoring-checks.md — 完了前チェック・脚注・overflow

スライドを書き終えたら必ずここに立ち戻る．SKILL.md は「完了宣言前にこのファイルを開け」とだけ指示している．

## 完了前チェックリスト

執筆ワークフロー step 5 で順に行う：

1. `quarto render posts/<slug>/index.qmd` を実行．
2. ブラウザでスライドを開き **全スライド** を順送りして overflow を目視．スクロールバーが出ているスライドは overflow している．
3. PDF 出力する deck であれば `reveal2pdf.sh` で生成して PDF も確認．**PDF では truncate されて情報が消える**．

## 脚注（footnote）の書き方

脚注は **anchor をスライド本文に，定義は同スライドの末尾にまとめて置く．** スライドごとに定義を集約する．

**スライド本文（anchor のみ）：**

```markdown
- CLAUDE.md[^claudemd]は[**セッション開始時**]{.regmonkey-bold}に必ず読み込まれる
```

**同スライドの末尾：**

```markdown
... 最後の行 ...

<!-- footer -->

[^claudemd]: Claude Code が読むのは `CLAUDE.md`．他ツールで採用される `AGENTS.md` ではない点に注意．
```

- `<!-- footer -->` HTML コメントを区切りに置く．後で全脚注定義を grep / 編集するときの目印．
- Quarto Reveal はこの定義を解釈して anchor のあるスライド下端に脚注番号付きで描画する．anchor の隣に定義を書く必要はない．
- スライド途中（あるスライドの直下など）に脚注定義を置くと HTML 出力が崩れる場合がある．**必ず同スライドの末尾に集約**．

## スライド height overflow を防ぐ

キャンバスは **1600 × 900** 固定．H2（テイクアウェイ）・`h2-submessage`・フッター・ページ番号で約 250px を消費するため，**本文の作業領域は約 650px** しか残らない．Reveal.js は overflow をスライド内スクロールにしてしまうため，PC ブラウザ表示ではパッと気づきにくく，PDF 出力（`reveal2pdf.sh`）では truncate されて情報が消える．

### overflow を起こしやすいパターン

- **info-box（上段）+ 2列 `square-box-500`（下段）の縦積み．** 合計 600px 超で `h2-submessage` があると確実にあふれる．→ `square-box-500` を `-450`/`-400` に下げるか，上段 info-box を `h2-submessage` に統合する．
- **第一階層の箇条書きが 7 項目以上．** MMUF に立ち返ると不要項目が混じっていることが多い．6 項目以下に削るか，2 スライドに分割する．
- **箇条書きのネストが 3 段以上．** 子項目を整理して 2 段以内にする．3 段は本当に必要か再考する．
- **`h2-submessage` が 2 行に折り返す．** 33 字以内に削るか，本文の冒頭1項目として吸収する．折返しは上端の余白を奪う．
- **大きなコードブロック + 解説の併存．** コード側を `.font-09` 以下にするか，解説を別スライドに分離する．

### 対処の優先順位（上から順に試す）

1. **削る．** MMUF に戻り H2 を支えない項目を消す．多くの overflow は「言いたいことが2つある」サインで，分割か削減で解決する．
2. **分割する．** 1スライドに2メッセージを抱えていないか確認する．抱えているなら `#` で章扉を増やすか H2 を分ける．
3. **密度を下げる．** ブロックに `.font-09`/`.lh-12` を付けてタイポグラフィを締める．`.font-08` まで下げると本文として読みづらくなるので注意．
4. **ボックスサイズを変える．** `square-box-500` → `-450`/`-400` で縦を圧縮．逆にコンテンツがどうしても必要なら `-600`/`-700` に拡張する．
5. **最後の手段．** `style="font-size: 0.85em"` をブロックに直接付与．読みやすさを犠牲にするので，他の選択肢を試した後に限る．
